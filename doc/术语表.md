# 术语表

术语	定义
本体（Ontology）	对特定领域知识的形式化表示，包含概念、属性、关系和约束的完整定义体系，是企业数字化知识的基础框架
对象类型（Object Type）	定义业务实体类型的模式（Schema），描述该类实体拥有的属性、关联关系和可执行动作的完整结构
对象实例（Object Instance）	对象类型的具体数据记录，代表现实世界中一个具体的业务实体（如「阿里巴巴集团」是「企业客户」对象类型的一个实例）
关联类型（Link Type）	定义两种对象类型之间关系的模式，包含基数约束（1:1/1:N/M:N）、方向性和语义描述
对象集合（Object Set）	通过条件筛选动态计算或静态快照得到的对象实例集合，动态集合实时更新，静态集合记录某时刻快照
动作类型（Action Type）	封装对本体执行有状态操作的配置单元，包含输入参数定义、前置条件校验和执行逻辑规则
接口（Interface）	定义多个对象类型共同遵循的抽象属性约定，实现跨类型统一访问，类似面向对象编程中的接口概念
命名空间（Namespace）	用于隔离和组织本体资产的逻辑分区，支持按业务域（如 business_core、hr_system）或团队划分管理边界
数据血缘（Data Lineage）	追踪数据从源头系统经过加工处理到最终本体对象的完整链路，支持影响分析和合规审计
RBAC	基于角色的访问控制（Role-Based Access Control），通过为用户分配角色来管理系统资源的访问权限
Webhook	事件驱动的 HTTP 回调机制，当 Action 执行等特定事件发生时，系统主动向预设端点推送通知消息
MDM	主数据管理（Master Data Management），对企业核心业务实体（客户/产品/员工等）进行集中化标准治理
ETL	数据抽取（Extract）、转换（Transform）、加载（Load）的数据处理流程，用于将源系统数据同步至本体对象实例
